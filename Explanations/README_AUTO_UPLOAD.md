# 自动上传到GitHub脚本说明

本项目提供了两个自动将文件上传到GitHub的脚本，可以帮助您每天自动提交和推送代码更改。

## 脚本类型

### 1. 批处理脚本 (`auto_upload_to_github.bat`)
- **优点**：简单易用，不需要Python环境
- **缺点**：错误处理相对简单

### 2. Python脚本 (`auto_upload_to_github.py`)
- **优点**：功能更丰富，有更好的错误处理和日志记录
- **缺点**：需要安装Python环境

## 使用方法

### 前提条件
1. 您的项目已经初始化了Git仓库
2. 您已经将本地仓库与GitHub远程仓库关联
3. 您的Git已经配置了正确的用户名和邮箱
4. 您的Git已经设置了SSH密钥或记住了密码，避免每次推送都需要输入凭据

### 手动运行脚本

#### 批处理脚本（还未更新）
1. 双击运行 `auto_upload_to_github.bat` 文件
2. 或者在命令行中执行：`auto_upload_to_github.bat`

#### Python脚本（最新版）
1. 确保您已经安装了Python
2. 在命令行中执行：`python auto_upload_to_github.py`

### 设置Windows定时任务（自动每天运行）

以下是设置Windows定时任务的步骤：

1. 打开"任务计划程序"：
   - 按下 `Win + R`，输入 `taskschd.msc`，然后按回车

2. 创建基本任务：
   - 在右侧"操作"面板中，点击"创建基本任务..."

3. 配置任务名称和描述：
   - 输入任务名称（例如："每天自动上传到GitHub"）
   - 输入描述（可选）
   - 点击"下一步"

4. 设置触发器：
   - 选择"每天"，然后点击"下一步"
   - 设置开始时间（例如：23:00）
   - 设置重复任务间隔（1天）
   - 点击"下一步"

5. 设置操作：
   - 选择"启动程序"，然后点击"下一步"
   - 点击"浏览"，找到您要使用的脚本文件（.bat或.py）
   - 如果选择Python脚本，需要在"程序/脚本"中输入Python可执行文件的完整路径，在"添加参数"中输入脚本的完整路径
   - **示例**：
     - 程序/脚本：`C:\Users\29606\AppData\Local\Programs\Python\Python314\python.exe`
     - 添加参数：`C:\Users\29606\codingFile\auto_upload_to_github.py`
   - 点击"下一步"

6. 完成设置：
   - 勾选"当单击完成时，打开此任务属性的对话框"（可选）
   - 点击"完成"

7. 高级设置（可选）：
   - 如果勾选了上一步的选项，会打开属性对话框
   - 在"常规"选项卡中，可以设置"不管用户是否登录都要运行"（需要管理员权限）
   - 在"触发器"选项卡中，可以设置更多高级定时选项
   - 点击"确定"保存设置

## 日志记录

两个脚本都会在您的本地目录下生成一个日志文件 `auto_upload_log.txt`，记录每次运行的详细信息，包括：
- 运行时间
- 各个步骤的执行结果
- 任何错误信息

您可以查看这个日志文件来了解脚本的执行情况。

## 注意事项

1. 确保您的GitHub远程仓库存在，并且本地仓库已经与远程仓库关联
2. 如果您使用HTTPS协议连接GitHub，可能需要设置Git记住密码，或者使用SSH协议
3. 如果脚本运行失败，请查看日志文件了解详细的错误信息
4. 建议先手动运行脚本测试，确保一切正常后再设置定时任务
5. 确保您的计算机在定时任务运行时处于开机状态，并且可以访问互联网

## 如何检查Git配置

您可以在命令行中执行以下命令来检查Git配置：

```bash
# 检查用户名
git config --global user.name

# 检查邮箱
git config --global user.email

# 检查远程仓库关联
git remote -v

# 设置记住密码（HTTPS协议）
git config --global credential.helper store
```

## 如何生成SSH密钥

如果您想使用SSH协议连接GitHub，可以按照以下步骤生成SSH密钥：

1. 打开命令行
2. 执行以下命令（替换为您的GitHub邮箱）：
   ```bash
   ssh-keygen -t ed25519 -C "your_email@example.com"
   ```
3. 按照提示完成密钥生成
4. 将公钥（通常是 `id_ed25519.pub` 文件的内容）添加到您的GitHub账户中

## 常见问题

### 脚本执行后没有任何提交
- 请确保您的项目目录中有文件被修改
- 检查 `.gitignore` 文件，确保没有忽略您想要提交的文件

### 推送失败
- 检查您的网络连接
- 确保您的GitHub账户有推送权限
- 检查远程仓库地址是否正确

### 定时任务没有执行
- 检查任务计划程序的设置是否正确
- 确保您的计算机在定时时间处于开机状态
- 查看Windows事件查看器，了解任务执行失败的原因

如果您遇到其他问题，请查看日志文件获取详细信息，或者在GitHub上寻求帮助。
