# Windows 自动 Git 脚本说明

## 1. 脚本概述

本目录包含两个用于 Git 仓库管理的自动脚本：
- `auto_update_git.py`：智能 Git 仓库自动更新脚本
- `auto_upload_to_github.py`：Git 仓库自动上传脚本

这些脚本能够帮助您自动化 Git 仓库的日常管理工作，提高工作效率。

## 2. 核心功能

### 2.1 自动更新脚本 (`auto_update_git.py`)

- **智能同步判断**：精确检测仓库与远程分支的同步状态
- **本地修改保护**：自动保存和恢复本地未提交的修改
- **冲突处理机制**：在发生冲突时提供清晰的解决方案
- **详细日志记录**：记录所有操作过程，便于问题排查
- **自动错误处理**：在各种异常情况下保持仓库一致性

### 2.2 自动上传脚本 (`auto_upload_to_github.py`)

- **自动状态检查**：检测仓库的修改状态
- **自动提交功能**：根据当前日期时间生成提交信息
- **自动推送到远程**：将本地提交推送到 GitHub
- **日志记录**：记录所有上传操作过程

## 3. 适用环境

- **操作系统**：Windows
- **依赖软件**：Git、Python
- **Python 版本**：Python 3.6 及以上

## 4. 安装与配置

### 4.1 克隆或下载脚本

将脚本文件下载到 Windows 系统的任意目录，例如：
```
c:\Users\29606\codingFile\Scripts\
```

### 4.2 配置脚本

使用文本编辑器（如记事本、VS Code）打开脚本文件，根据需要修改以下参数：

#### 自动更新脚本配置

```python
# 项目配置
project_dir = "c:\Users\29606\codingFile"  # 你的项目目录路径
log_path = os.path.join(project_dir, "Log_Files", "auto_update_log.txt")  # 日志文件路径
```

#### 自动上传脚本配置

```python
# 项目配置
project_dir = "c:\Users\29606\codingFile"  # 你的项目目录路径
log_path = os.path.join(project_dir, "Log_Files", "auto_upload_log.txt")  # 日志文件路径
```

- `project_dir`：设置为你的 Git 仓库所在的目录路径
- `log_path`：设置日志文件的保存路径（默认保存在项目目录下的 Log_Files 文件夹）

6. 浏览并选择 python.exe 文件
7. 在「添加参数」中输入脚本路径：
   ```

   ```
8. 完成创建

## 6. 工作原理

### 6.1 自动更新脚本工作流程

1. **检查 Git 状态**：检测仓库的当前状态
2. **判断同步状态**：
   - 检查是否有未提交的修改或未跟踪的文件
   - 获取远程仓库的最新更新
   - 比较本地分支与远程分支的差异
3. **保存本地修改**：如果有本地修改，自动保存
4. **执行更新操作**：执行 `git pull` 命令更新仓库
5. **恢复本地修改**：更新完成后，自动恢复之前保存的本地修改

### 6.2 自动上传脚本工作流程

1. **检查 Git 状态**：检测仓库的当前状态
2. **添加修改文件**：将所有修改的文件添加到暂存区
3. **提交修改**：根据当前日期时间生成提交信息并提交
4. **推送到远程**：将本地提交推送到 GitHub

## 8. 常见问题与解决方案

### 8.1 问题：脚本执行失败，提示 "'git' 不是内部或外部命令"

**解决方案**：确保 Git 已安装并添加到系统环境变量中
1. 下载并安装 Git：https://git-scm.com/download/win
2. 安装时选择 "Add Git to the system PATH"

### 8.2 问题：Python 命令无法识别

**解决方案**：确保 Python 已安装并添加到系统环境变量中
1. 下载并安装 Python：https://www.python.org/downloads/
2. 安装时选择 "Add Python to PATH"

### 8.3 问题：更新失败，提示权限不足

**解决方案**：确保对 Git 仓库有读写权限

### 8.4 问题：本地修改恢复失败，提示冲突

**解决方案**：手动解决冲突
```bash
# 查看保存的修改
git stash list

# 应用保存的修改（不删除）
git stash apply stash@{0}

# 手动解决冲突后，提交修改
git add .
git commit -m "Resolve conflict"
```

### 8.5 问题：上传失败，提示 "fatal: not a git repository"

**解决方案**：确保在 Git 仓库目录下运行脚本，或正确配置脚本中的 `project_dir` 参数

## 9. 注意事项

1. **网络连接**：确保系统有稳定的网络连接，以便与远程仓库通信
2. **Git 配置**：确保已正确配置 Git 用户信息（用户名和邮箱）
   ```bash
   git config --global user.name "Your Name"
   git config --global user.email "your.email@example.com"
   ```
3. **分支管理**：脚本默认使用 `main` 分支，如需使用其他分支请修改脚本中的分支名称
4. **磁盘空间**：确保有足够的磁盘空间来存储更新内容
5. **定时任务**：设置定时任务时，确保脚本路径正确且有执行权限
